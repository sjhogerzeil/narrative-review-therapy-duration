# blog-therapy-duration

Annotated literature review on the duration and trajectory of healing from complex developmental trauma.

## Project structure

```
blog-therapy-duration/
├── CLAUDE.md
├── plan.md                     → pointer to 2_methods/ (single source of truth)
├── 1_introduction/             → background, population, research questions
├── 2_methods/                  → search strategy, annotation protocol, synthesis method
├── 3_results/
│   ├── sources/                → annotated source notes (one .md per source)
│   ├── primary/                → extracted data by primary synthesis dimension
│   └── secondary/              → extracted data by secondary question
├── 4_discussion/               → synthesis, limitations, implications
├── 5_tables_and_figures/       → summary tables, timelines
├── 6_appendix/
│   └── iterations/             → re-run documentation when new questions emerge
├── 7_references/
│   └── references.md           → APA 7 bibliography (auto-generated by sync.sh)
├── 99_search/
│   └── index.md                → semantic search index (auto-generated by sync.sh)
└── sync.sh                     → keeps references, index, and READMEs in sync
```

## Conventions

- **Source files** live in `3_results/sources/[layer-dir]/[author-year-slug].md`
- **Source template** is at `3_results/sources/SOURCE_TEMPLATE.md`
- **YAML frontmatter** on source files allows relevance scanning without reading the body
- **Back-references** use source IDs: `[author-year-slug]`
- **APA 7** citations in all source notes
- **"No data" is a finding** — always note absence explicitly in secondary questions and cross-cutting observations

## Stance

Honest advocacy: the working thesis is that healing takes years. The review builds the case while steel-manning counterarguments. See `1_introduction/5_stance.md`.

## Key files to read first

1. `1_introduction/4_research_questions.md` — primary + 6 secondary questions
2. `2_methods/1_design.md` — overall approach
3. `2_methods/2_search_strategy.md` — layers, inclusion/exclusion, stopping rules
4. `3_results/sources/SOURCE_TEMPLATE.md` — annotation format
5. `2_methods/12_rerun_protocol.md` — what to do when new questions emerge

## Sync

After annotating sources, run `./sync.sh` to update:
- `7_references/references.md` — bibliography from source YAML `citation` fields
- `99_search/index.md` — semantic index from all YAML headers
- Layer READMEs — annotated sources tables

Use `./sync.sh check` for a dry run to find missing citations or unindexed sources.
