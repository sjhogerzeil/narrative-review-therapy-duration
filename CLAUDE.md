# blog-therapy-duration

Structured narrative review (CIS methodology) on the duration and trajectory of healing from complex developmental trauma.

## Quick orientation for AI agents

**Read order for any agent entering this project:**

1. This file (CLAUDE.md) — project structure, conventions, task routing
2. `7_process/progress.md` — where are we? what's next? what's waiting for human input?
3. `_index/index.md` — scan YAML descriptions to find relevant files without reading them all
4. Only then read specific files needed for your task

**Task routing — what to read based on your task:**

| Task | Read these files |
|------|-----------------|
| Search for sources in a specific layer | `2_methods/[layer file]` + layer `README.md` in `3_results/sources/` |
| Annotate a source | `3_results/sources/SOURCE_TEMPLATE.md` + `2_methods/9_annotation_protocol.md` |
| Extract data into results | `2_methods/11_synthesis_method.md` + relevant `3_results/primary/` or `secondary/` file |
| Check what's been done | `_index/index.md` (scan status fields) + layer READMEs (annotated sources tables) |
| Understand the research questions | `1_introduction/4_research_questions.md` |
| Understand what population we're studying | `1_introduction/2_population.md` |
| Add a new question/dimension | `2_methods/12_rerun_protocol.md` |
| Handle an inaccessible source | `2_methods/9_annotation_protocol.md` § Inaccessible sources + `7_process/todo.md` |
| Write discussion content | `4_discussion/[section].md` — check `depends_on` in YAML header |

**After completing any task:**
1. Append meta-commentary to `7_process/execution_log.md` (issues, questions, emerging patterns)
2. Run `./_scripts/sync.sh all` if you created or modified source notes
3. If you added a source, also add its citation to `6_references/references.md` (or let sync handle it)

## Conventions

- **Source files** live in `3_results/sources/[layer-dir]/[author-year-slug].md`
- **Layer 1 sublayers:** trauma-clinical (1a), psychoanalytic (1b), humanistic-existential (1c), neuroscience (1d) — each has its own subdirectory
- **YAML frontmatter** on all files allows relevance scanning without reading the body
- **Back-references** use source IDs: `[author-year-slug]`
- **APA 7** citations in all source notes
- **"No data" is a finding** — always note absence explicitly
- **Append-only** — source notes and extraction tables are never rewritten, only appended (see `2_methods/9_annotation_protocol.md`)
- **Git captures chronology** — the final state of every file is the auditable record; git history shows how it evolved

## Stance

Honest advocacy: the working thesis is that healing takes years. The review builds the case while steel-manning counterarguments. See `1_introduction/5_stance.md`.

## Language and register

**English academic paper.** Write in academic English — clear, precise, well-structured prose. Not informal, not jargon-heavy. A Dutch blog post will be written later based on this report (separate project, different voice).

## Methodology

Structured narrative review using critical interpretive synthesis (CIS; Dixon-Woods et al., 2006). See `2_methods/1_design.md` for full methodological framework.

## Agent execution protocol

1. **Before starting:** Read this file + `_index/index.md` to orient
2. **During execution:** Follow the task routing table above
3. **After completing:** Append to `7_process/execution_log.md`:
   - Issues identified during execution
   - Questions raised that the plan doesn't answer
   - Suggestions for plan improvement
   - Emerging patterns that don't fit existing dimensions
   - Inaccessible sources encountered (also add to `7_process/todo.md`)
4. **Run sync:** `./_scripts/sync.sh all` after creating/modifying source notes

## Orchestrator review protocol

After each execution batch, the orchestrator reviews `7_process/execution_log.md` for:
- Recurring problems → structural gap in the plan
- New questions from multiple agents → missing secondary question
- Emerging patterns → potential synthetic construct (see `2_methods/11_synthesis_method.md` § Emergent synthetic constructs)
- Plan drift → adjust stopping rules or scope
- Accumulating inaccessible sources → systematic gap assessment

## Sync

After annotating sources, run `./_scripts/sync.sh` to update:
- `6_references/references.md` — bibliography from source YAML `citation` fields
- `_index/index.md` — semantic index from all YAML headers
- Layer READMEs — annotated sources tables

Use `./_scripts/sync.sh check` for a dry run to find missing citations or unindexed sources.
